DESIGN DECISIONS (2/2)

설계 결정: 실행 + 보안 + 인프라

동시성 단순화부터 이식성 증명까지. 24가지 설계 결정 완료.

실행 + 동시성

SessionLane per-key Sem(1) vs 4 hardcoded lanes

같은 key 직렬, 다른 key 병렬. C3 scheduler race, H2 announce race 구조적 해소.

CoalescingQueue 삭제 vs 유지 (no harm)

callback=no-op, dedup=0. 동기 루프 구조에서 250ms 재제출 자체가 불가능. Dead code 삭제.

Structured Events (30 types) vs Raw Console Stream

Client-side 렌더링 = spinner, 병렬 tool 추적 가능. Server는 상태만 관리.

time_budget_s vs max_rounds

시간 기반 = 모드별 예산 차등 가능. DAEMON 120s, SCHEDULER 300s, IPC unlimited.

보안 + 인프라

PolicyChain AND 6-layer vs OR 허용

모든 정책 통과 필수. Org(priority=5)이 하위 규칙 오버라이드.

Swiss Cheese 5-Layer vs 단일 종합 검사

독립 실패 모드 AND-chain. 69.4%→99.8%. BiasBuster + Cross-LLM + Calibration.

서버 사이드 점수 계산 vs LLM composite_score

(axes_sum-8)/32x100 투명 계산. LLM 수치는 비교 참고만.

이식성 증명: GEODE → REODE

동일 4-Layer 코어. 도메인 플러그인만 교체. core 변경 0줄. 83/83 빌드 통과.
