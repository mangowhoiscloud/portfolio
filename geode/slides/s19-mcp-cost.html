MCP 단순화 — Before → After

BEFORE: 3-Layer 병목

  • MCPRegistry (257줄) — LLM 탐색을 가로막는 중앙 레지스트리

  • Catalog — Registry에 종속, 독립 검색 불가

  • Proxy stub 9개 — 중복 래퍼, 유지 관리 부담

AFTER: 2-Layer 직관

  • MCPRegistry 삭제 → Catalog 검색 전용으로 축소

  • config.toml [mcp.servers]로 설정 통합

  • 44개 서버 auto-discovery · Proxy stub 9개 제거

Trade-off — 제어 vs 탐색 자유도

Registry는 도구 목록을 완전 통제하지만 LLM의 런타임 탐색을 막습니다. Catalog로 전환하면 44개 서버를 동적 관리하면서도 HITL 4-Tier 권한 모델로 실행 경계를 유지합니다.
