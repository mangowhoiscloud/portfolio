C1. Phase 순서가 중요한 이유 (S18 보강)

문제: skill을 memory보다 먼저 주입하면 스킬의 context:fork가 오래된 메모리를 참조합니다. memory를 template보다 먼저 주입하면 메모리가 시스템 프롬프트를 오버라이드합니다.

6-Phase 순서 근거

1

template

시스템 정체성. 다른 모든 것의 기반이 됩니다.

2

override

도메인별 설정. template 위에 얹어야 우선순위 명확.

3

skill

런타임 스킬 42개. 3-tier lazy loading. override 후에야 scope 결정 가능.

4

memory

5-Tier 메모리. skill 이후여야 스킬이 최신 메모리를 참조.

5

bootstrap

런타임 상태 주입. 모든 정적 설정 완료 후 동적 상태 합류.

6

SHA-256 assemble

최종 해시. 변경 부분만 해시가 달라져 cache hit rate 보존.
