PART 0 · THESIS

확률적 시스템(LLM)은 제어 없이 발산합니다

Harness 4-Axis: 장기 운용에서 모두가 시간에 따라 열화한다

핵심 주장

LLM 단독 실행은 비결정론, 컨텍스트 손실, 비용 폭발, 루프 고착 4가지 이유로 반드시 발산합니다.

발산을 수렴시키는 것이 하네스의 역할입니다. 4개 축이 없으면 자율 실행은 불가능합니다.

장기 운용의 문제

4축 모두가 시간에 따라 열화합니다.

  • 컨텍스트 포화 → 정보 손실
  • 루프 고착 → 무한 반복
  • 비용 누적 → 예산 초과
  • 관측 공백 → 드리프트 미감지

→ 이를 자동 보정하는 것이 하네스의 본질

축 1 · Context Control

컨텍스트 제어

PromptAssembler 6-Phase 조립 + SHA-256 캐싱으로 정보 주입을 설계합니다. 장기 세션에서 80%/95% 임계값 compaction이 자동 발동합니다.

PromptAssembler · CompactionHook

축 2 · Execution Loop

실행 루프

while(tool_use) 50 rounds × 200 turns. 비용 초과·수렴 고착·탐색 고착을 3개 자동 브레이크로 차단합니다.

AgenticLoop · SafetyBrake

축 3 · Verification

검증

비결정론을 신뢰로 전환. 5-Layer 검증(Guardrails→BiasBuster→Cross-LLM→Rights→Ground Truth)으로 단일 LLM 편향을 차단합니다.

VerificationPipeline · CrossLLM

축 4 · Observability

관측

46 Hook Events가 신경계처럼 전파. TokenTracker로 실시간 비용 관측, CUSUM 드리프트 감지로 열화를 즉시 포착합니다.

HookSystem · TokenTracker

4축은 서로 보완 — 하나의 결함이 전체 자율 실행을 무너뜨립니다
