결정론적·제로 레이턴시 라우팅부터 도메인 이식성까지 — 8가지 인프라 결정

게이트웨이 · 통신

정적 바인딩 라우팅 vs LLM 의도 라우팅

결정론적·제로 레이턴시·감사 가능 — LLM 라우팅은 비용·지연·불확실성 추가

Daemon Thread 폴링 vs asyncio 이벤트 루프

동기 LangGraph와 단순 통합 — 비동기 복잡도 없이 동시성 확보

Echo Prevention (subtype + _last_ts) vs DB dedup

경량·DB 의존 없음 — 3초 폴링 주기에 충분한 메모리 기반 dedup

계층형 세션 키 vs 플랫 UUID

gateway:ch:id:sender:thread — 스레드별 컨텍스트 격리, 사람이 읽을 수 있는 키

인프라 · 이식성

DomainPort Protocol + ContextVar DI vs ABC 상속

구조적 타이핑 — 상속 결합 없음, 스레드 안전 교체 가능

SqliteSaver + MemorySaver fallback vs 항상 인메모리

프로덕션 크래시 복구 + 개발 경량 — 단일 팩토리로 환경별 전환

서버 사이드 점수 계산 vs LLM composite_score

(axes_sum-8)/32×100 — 투명·검증 가능, LLM 수치는 비교 참고만

Serve = REPL 팩토리 + hitl_level=0 vs 별도 코드베이스

_build_agentic_stack() 재사용 — 동작 플래그만 다름, 코드 중복 0
