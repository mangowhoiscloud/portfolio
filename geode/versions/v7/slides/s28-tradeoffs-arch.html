핵심 Trade-off 8선: 포기하지 않은 설계 결정은 결정이 아니다

아키텍처, 에이전트 실행, 인프라 전반. 모든 피봇은 CHANGELOG에 “왜”가 기록되어 있다

Port/Adapter DI

직접 호출 단순성을 포기하고 DI를 도입했더니 GEODE→REODE 이식 시 코어 코드 변경 0줄이 됐다.

Lossless Context Selection

단순 요약(빠르지만 정보 손실)을 버리고 LLM이 직접 청크를 선별하게 했더니 비용 0.03%로 절감되면서 정보 손실이 사라졌다.

5-Tier Safety: 병렬 vs 순차

T3-4 병렬 처리 속도를 포기하고 DANGEROUS 명령은 개별 순차 승인으로 격리했더니 권한 에스컬레이션 경로가 차단됐다.

Sub-Agent 4096 Token Guard

자식 출력 전체 반환(완전한 컨텍스트)을 포기하고 4096 guard를 적용했더니 부모 컨텍스트 폭발이 방지되어 MAX_CONCURRENT=5 병렬이 가능해졌다.

SHA-256 PromptAssembler

수동 편집의 즉각성을 포기하고 6-Phase 조립과 SHA-256 추적을 도입했더니 cache_control ephemeral로 2회차 입력 비용이 10%로 줄었다.

MCPRegistry 삭제

중앙 집중 도구 관리(단일 조회 포인트)를 포기하고 auto-discovery를 채택했더니 proxy stub 9개가 제거되고 45 MCP 서버를 자동 등록할 수 있게 됐다.

Hook Priority 핸들러 순서 보장

병렬 핸들러 처리량을 포기하고 Priority 정렬을 넣었더니 Security 핸들러가 Logging보다 항상 먼저 실행되는 것이 보장됐다.

50 Rounds × 200 Turns 상한

완전 자율(무제한 실행)을 포기하고 수렴 감지 + 비용 상한을 도입했더니 파이프라인 폭주가 차단되고 장기 운용 비용이 예측 가능해졌다.
