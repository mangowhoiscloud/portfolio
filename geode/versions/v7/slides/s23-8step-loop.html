Socratic Gate (①)

구현 전 반드시 통과해야 하는 5가지 질문:

  • 왜 이 기능이 필요한가?
  • 어떤 실패 모드가 있는가?
  • Trade-off는 무엇인가?
  • 테스트 가능한가?
  • CHANGELOG에 이유를 쓸 수 있는가?

이 5가지를 답하지 못하면 코드 작성 불가이며 루프가 ①로 되돌아간다.

왜 8단계인가?

5단계(Plan→Implement→Test→Merge→Deploy)면 충분하지 않은가?

5단계 부재 시

GAP Audit·Socratic Gate 없으면 불필요한 구현이 50%에 달하며 잘못된 전제로 PR이 병합된다
