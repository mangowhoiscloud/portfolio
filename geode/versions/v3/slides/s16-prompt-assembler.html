SHA-256 변경 추적

조립 후 SHA-256 해시 생성 — 동일 프롬프트 재조립 시 캐시 히트 판별. 수동 편집으로는 불가능했던 변경 추적을 코드 레벨에서 보장한다.

hash = sha256(assembled_prompt) → cache_key

Trade-off: 조립 복잡도 vs 캐시 안정성

Before — 수동 편집

  • 변경 추적 불가 — 캐시 미스 빈발
  • 프롬프트 버전 관리 없음

After — 6-Phase 조립

  • SHA-256 추적 + 캐시 히트 보장
  • 2회차부터 입력 비용 10%
