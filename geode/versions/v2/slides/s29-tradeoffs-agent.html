Trade-offs — Agent + Security + Gateway 설계 결정

자율성과 안전성 사이, 각 결정의 기각된 대안과 채택 근거

채택

3-Provider Circuit Breaker

vs

기각: 단일 Provider

Rate Limit·장애 시 파이프라인 전체 중단. Circuit Breaker 5회 실패→60s OPEN→Half-Open으로 자동 복구

채택

Exponential Backoff ±25% Jitter

vs

기각: 고정 지연 재시도

고정 지연은 다수 에이전트가 동시 재시도 → Thundering Herd. Jitter로 재시도 분산, RateLimit 회피

채택

ZhipuAI (GLM-5) 3번째 Provider

vs

기각: Gemini

Gemini는 API 응답 포맷이 비표준 케이스 多. ZhipuAI는 OpenAI-compatible API로 Adapter 재사용 가능

채택

46-Event Hook Observer

vs

기각: 하드코딩 콜백

하드코딩 콜백은 이벤트 추가 시 코어 수정 필요. Priority 핸들러 + 에러 격리로 확장성과 격리 확보

채택

Dual Trust (Input + Output Gate)

vs

기각: Input만 검사

Input만 막으면 Output 경로로 secret 우회 가능. Output Validation 독립 레이어로 양방향 차단

채택

Deny-by-default (serve 모드)

vs

기각: allow-by-default

HITL 없는 headless 모드에서 WRITE 허용이면 의도치 않은 파일 수정. 최소 권한 원칙 → SAFE만 허용

채택

MCP-first + fixture fallback

vs

기각: 라이브 전용

라이브 전용이면 CI 환경에서 테스트 불가. fixture fallback으로 오프라인 테스트 + 라이브 수집 양립
