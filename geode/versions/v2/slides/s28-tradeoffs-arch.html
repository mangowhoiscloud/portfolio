각 결정에 기각된 대안이 있었다. 채택 이유와 결과를 검증한다.

채택

LangGraph StateGraph DAG

vs

기각: 직접 함수 체인

분기/루프백이 있는 파이프라인은 함수 체인으로 상태 관리 불가 → StateGraph로 노드 재진입과 조건 분기 표현

채택

Port/Adapter DI 패턴

vs

기각: 직접 의존성

직접 의존성이면 LLM Provider 교체 시 파이프라인 전체 수정 → Port 추상화로 Provider 교체 시 Adapter만 교체

채택

6-Layer Verification

vs

기각: 단일 LLM 평가

단일 평가자는 Confirmation Bias. Guardrails→BiasBuster→Cross-LLM→Rights Risk→GT 5단계 체인으로 69.4→99.8 신뢰도

채택

Lossless Selection 압축

vs

기각: Lossy 요약

요약은 코드/수치 정보 손실. LLM이 코드로 관련 청크 선별 → 정보 손실 0%, 비용 0.03%

채택

6-Phase PromptAssembler

vs

기각: 수동 프롬프트 편집

수동 편집은 변경 추적 불가, 캐시 미스. SHA-256 추적 + ephemeral 캐시 → 2회차부터 입력 비용 10%

채택

4096-token 서브에이전트 Guard

vs

기각: 무제한 반환

서브에이전트 결과 무제한 반환 시 부모 컨텍스트 폭발. 4096-token guard로 구조화 요약 강제

채택

asyncio.gather 5-Tier 병렬

vs

기각: 순차 실행

순차 실행은 도구 N개 × 지연 선형 증가. SAFE/AUTO는 병렬, WRITE 이상은 순차로 안전성과 속도 균형
