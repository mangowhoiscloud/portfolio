장기 자율 실행의 3대 위험과 자동 브레이크

자율성과 안전의 균형 — 사람 없이 하네스를 보호하는 3가지 메커니즘

1

비용 폭발

위험 — 실제 비용 계산

while(tool_use) 특성상 비용 폭발은 구조적으로 불가피. 50라운드 × $0.15/turn = 세션당 $7.50. budget 초과 시 자동 정지가 없으면 하룻밤 무인 실행에 $500+ 소모 가능.

자동 브레이크 — 인과 흐름

TokenTracker가 매 턴 비용 누적 → budget 상한 도달 시 budget_stop 신호 → LOOP_END 강제 발화. 사람 없이 비용 상한을 보장합니다.

Karpathy P3 원칙 적용

budget_stop → LOOP_END

2

무한 루프

위험 — 수렴 고착의 비용

LLM이 동일한 에러를 반복하면 50 rounds 전체를 낭비. 동일 요청에 $7.50를 태우고 실패를 반환합니다.

자동 브레이크 — 인과 흐름

동일 에러 3회 감지 → 수렴 판정 → Haiku→Sonnet→Opus 상위 모델 에스컬레이션 → 모델 체인 소진 시 Cross-Provider 자동 이동.

Karpathy P4 원칙 적용

convergence → model escalation

3

탐색 고착

위험 — 탐색 편향의 결과

동일 도구만 반복 호출하면 탐색 공간이 좁아지고, 다른 접근법으로 해결 가능한 문제를 50 rounds 내내 놓칩니다.

자동 브레이크 — 인과 흐름

도구 반복 5회 감지 → Hook이 hint 메시지 자동 주입 → LLM이 다른 도구 경로를 탐색. 다양성을 구조적으로 강제합니다.

다양성 강제 (Diversity Forcing)

tool_repeat_5x → hint_inject

Trade-off — 자율성 vs 안전: 브레이크가 강할수록 자율 실행 범위가 줄어듭니다. 3가지 임계값(budget 한도, 수렴 횟수 3회, 반복 횟수 5회)은 사용자가 조정 가능합니다. 기본값은 실험적으로 도출했습니다.
