Architecture Trade-offs

모든 설계 결정은 무언가를 포기하고 무언가를 얻는다 — Trade-off를 인식한 결정만이 장기 운용된다

Port/Adapter DI

포기: 직접 호출 단순성

얻음: GEODE→REODE 이식 시 코어 코드 변경 0줄

Lossless Context Selection

포기: 단순 요약(빠르지만 정보 손실)

얻음: LLM이 직접 선별 → 컨텍스트 비용 0.03%로 절감

Provider-aware Compaction

포기: 단일 압축 로직의 단순성

얻음: Anthropic Beta tool_use 캐시 규칙 준수 → Failover 안정성

6-Layer Architecture

포기: 초기 설계 속도 (레이어마다 인터페이스 정의)

얻음: 187 모듈 중 어느 레이어도 교체 가능한 구조

MCPRegistry 삭제

포기: 중앙 집중 도구 관리 (단일 조회 포인트)

얻음: Proxy stub 9개 제거 → 45 MCP auto-discovery

SHA-256 PromptAssembler

포기: 수동 편집의 즉각성

얻음: cache_control ephemeral → 2회차 입력 비용 10%로 감소

4-Tier Memory (휘발 → 불변)

포기: 단순 Redis 캐시 (단계 없는 단일 레이어)

얻음: Provider Failover 시에도 학습 내용 보존
