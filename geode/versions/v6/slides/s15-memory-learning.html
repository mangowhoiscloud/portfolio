왜 4-Tier인가?

단일 메모리면 프로젝트 간 격리가 안 되고, 세션 메모리만으로는 인사이트가 휘발된다. 4-Tier로 범위별 TTL과 접근 권한을 분리했다.

4-Tier 메모리가 LLMClientPort 추상화 위에 있기 때문에 Provider 전환 후에도 세션 학습 내용이 사라지지 않는다.

Trade-off: 자동 학습 오버헤드 vs 지식 휘발

PIPELINE_END마다 LLM 호출이 추가되지만, 세션마다 인사이트가 휘발되는 비용이 더 크다. write-back 덕분에 동일 분석 영역에서 세션이 누적될수록 품질이 개선된다.
