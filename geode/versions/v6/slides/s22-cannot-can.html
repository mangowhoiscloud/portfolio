Git 격리

CANNOT

Worktree 없이 코드 작업 금지

에이전트가 main 브랜치에 직접 쓰면 병렬 작업 충돌이 발생한다. 각 태스크는 반드시 독립 worktree에서 실행돼야 한다.

품질 게이트

CANNOT

lint/type/test 실패 상태 커밋 금지

실패한 상태의 커밋은 다음 에이전트를 오염시킨다. CI가 3종 체크를 모두 통과한 후에만 머지를 허용한다.

과잉 구현 방지

CANNOT

Plan 승인 없이 구현 착수 금지

LLM은 승인 없이 구현하려는 성질이 있다. 설계 문서에 인간 승인 서명이 없으면 첫 줄도 작성하지 않는다.

추적성

CANNOT

CHANGELOG 없이 기능 커밋 금지

500+ PR의 결정 근거가 사라지면 재설계 비용이 폭증한다. 모든 기능 커밋은 CHANGELOG 항목을 동반해야 한다.

html2pptx 제약

CANNOT

CSS gradient 금지

linear-gradient/radial-gradient는 html2pptx가 렌더링하지 못한다. 배경 효과는 solid color + border 조합으로 대체한다.

내러티브 품질

CANNOT

단순 박스형 배치 금지

키워드만 넣은 카드 그리드는 발표가 아니다. 각 박스는 "X 때문에 Y를 선택했고 Z가 나왔다" 수준의 인과 서술을 포함해야 한다.
