Scheduler + Gateway: 시간 기반 자율 실행과 채널 라우팅

Cron이 메인 컨텍스트를 오염시키는 문제 때문에 격리를 기본값으로 설계했고, 채널마다 어댑터를 새로 짜는 비용을 0으로 만들기 위해 Gateway를 단일 진입점으로 집중했다

Scheduler: 시간 기반 자율 실행

스케줄 유형

  • AT: 특정 시각 1회 실행
  • EVERY: 간격 반복 실행
  • CRON: cron 표현식 실행

NL Parser: "매일 오전 9시에 리포트 생성" → AT/EVERY/CRON으로 파싱

실행 흐름

NL 파싱 → action_queue 등록 → 트리거 시각 도달 → AgenticLoop 위임. 큐 관리로 동시 실행 충돌을 방지한다.

독립 실행

Cron 태스크가 메인 대화의 컨텍스트를 덮어쓰는 버그가 있었기 때문에 isolated=True를 기본값으로 바꿨다. 결과 주입이 필요하면 systemEvent로 메인에 선택적으로 전달한다.

채널별 AgenticLoop 인스턴스를 따로 관리하면 세션이 분산됐기 때문에 Gateway를 단일 진입점으로 집중했다. 채널 추가 시 어댑터 1개만 구현하면 된다.

Trade-off: Cron 격리로 메인 오염을 막았지만, 격리 세션은 이전 학습을 공유하지 않는다. systemEvent 주입으로 필요할 때만 연계한다. Gateway 단일화는 채널 다양성을 포기했지만 어댑터 1개로 신규 채널을 추가할 수 있게 됐다.
