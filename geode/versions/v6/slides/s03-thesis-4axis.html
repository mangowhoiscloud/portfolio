PART 0 · THESIS

확률적 시스템(LLM)은 제어 없이 발산한다

Harness 4-Axis: 장기 운용에서 컨텍스트·루프·검증·관측 4축이 없으면 열화가 불가피하다

핵심 주장

LLM 단독 실행은 비결정론, 컨텍스트 손실, 비용 폭발, 루프 고착 4가지 이유로 반드시 발산한다.

발산을 수렴시키는 것이 하네스의 역할이다. 4개 축이 없으면 자율 실행은 불가능하다.

장기 운용의 열화: 실제 예시

  • 컨텍스트 80% 채우면 응답 품질 저하 시작
  • 세션 10턴 넘으면 인사이트 휘발
  • Prompt drift로 행동 변화: 같은 입력에도 다른 출력 발생
  • 관측 공백 → CUSUM이 드리프트 미감지

→ 4축 자동 보정이 없으면 발산은 불가피

축 1 · Context Control

컨텍스트 제어

매 세션 프롬프트를 수동 편집하면 drift가 발생했기 때문에 PromptAssembler 6-Phase + SHA-256으로 자동 조립한다. 80%/95%에서 compaction이 발동해 루프가 컨텍스트 과부하로 중단되지 않는다.

PromptAssembler · CompactionHook

축 2 · Execution Loop

실행 루프

브레이크 없이 실행하면 비용·수렴·탐색 고착이 발생하기 때문에 3개 자동 브레이크를 설계했다. 실측 평균 4.2 라운드 종료, 상한은 50 라운드.

AgenticLoop · SafetyBrake

축 3 · Verification

검증

단일 LLM 평가는 Anchoring 편향으로 점수 분산이 ±18점이었기 때문에 5-Layer 검증(Guardrails→BiasBuster→Cross-LLM→Rights→Ground Truth)으로 신뢰도를 확보했다.

VerificationPipeline · CrossLLM

축 4 · Observability

관측

관측 공백이 있으면 CUSUM이 드리프트를 감지하지 못했기 때문에 46 Hook Events로 신경계를 구성하고 TokenTracker로 실시간 비용을 추적한다.

HookSystem · TokenTracker

4축은 서로 보완하며, 하나의 결함이 전체 자율 실행을 무너뜨립니다
