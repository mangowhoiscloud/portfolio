Frontier Mining: 4개 하네스 GAP 분석

자체 발명은 검증 비용이 높으므로, 먼저 읽고 3/4 수렴 규칙으로 차용 여부를 결정한다

Claude Code

while(tool_use) 패턴: LLM이 tool_use를 반환하는 한 루프를 유지하는 구조를 직접 확인한다. GEODE AgenticLoop 50 라운드 설계의 직접 근거가 된다.

채택: 루프 계속 조건 + end_turn 종료 패턴

autoresearch

Karpathy P1 제약 기반 설계, P4 래칫(되돌아가지 않음), P6 컨텍스트 예산 명시: 이 세 원칙이 CLAUDE.md 규칙 체계의 뼈대가 된다.

채택: P1/P4/P6 → CANNOT 규칙 3종

OpenClaw

Gateway, Session Key, Lane Queue, Plugin 4-계층 디스패치 구조에서 에이전트 요청을 분리된 채널로 라우팅하는 아이디어를 얻는다.

채택: SessionKey 격리 + LaneQueue 우선순위

Codex CLI

Worktree 격리, .owner 파일 보호, Dumb Platform 원칙으로 Git 충돌 없이 여러 에이전트가 병렬 작업하는 구조의 실증 사례다.

채택: Worktree 의무화 → CANNOT Git 규칙

왜 이 4개 하네스인가

Claude Code, autoresearch, OpenClaw, Codex CLI를 선택한 이유: 이 4개가 현존하는 프론티어 에이전트 하네스 중 가장 성숙하고, 소스 코드 또는 설계 문서가 공개되어 패턴을 추출할 수 있기 때문이다.

Trade-off: 자체 발명 vs 검증된 패턴

자체 발명은 독창적이지만 검증 비용이 높다. 프론티어 하네스에서 이미 검증된 패턴을 채택하면 설계 리스크를 줄인다.

3종 이상 합의된 패턴만 채택하고, 1종에서만 존재하는 패턴은 Socratic Gate Q5에서 재검증한다.
