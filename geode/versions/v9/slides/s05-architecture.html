Trade-off: 왜 6-Layer인가

3-Layer(Presentation/Business/Data)는 에이전트에 맞지 않는다. 도구, 메모리, 오케스트레이션이 독립적으로 교체되어야 하므로 6-Layer로 분리한다.

비용: 레이어 간 Protocol 정의가 필요하다.

결과: GEODE→REODE 피봇 시 L5만 교체하고 L0-L4는 그대로 유지한다.

왜 Hook을 L3에서 분리하는가

Hook을 L3 오케스트레이션에 두면 L0-L4 전체가 L3에 의존하는 역전 구조가 발생한다. 이를 막기 위해 Hook을 cross-cutting 모듈로 독립시킨다.

46개 이벤트가 L0-L5 모든 계층을 관측하되 어느 계층에도 종속되지 않는다.

교체 단위로서의 계층

  • LLM 교체 → L4만 변경
  • 메모리 백엔드 교체 → L5만 변경
  • 도구 추가 → L3만 확장
