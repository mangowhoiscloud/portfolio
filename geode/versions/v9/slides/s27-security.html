Security: Bash 3-Layer + Dual Trust Model

자율 에이전트는 신뢰 경계가 없으면 시스템을 파괴하기 때문에 PolicyChain이 실행 전 모든 명령을 검문한다

PolicyChain 다이어그램

Bash 3-Layer 실행 제어 — 인과 서술

L1: Blocked — Regex 9-Pattern Safe-First

rm‑rf, sudo, curl|sh 등 9개 regex 패턴을 우선 검사한다. safe‑first로 검사하면 ls / && rm‑rf / 같은 파이프 주입을 놓치기 때문에 저 순서가 필수다.

L2: Resource — setrlimit Child Process 강제

preexec_fn setrlimit으로 CPU 30s, FSIZE 50MB, NPROC 64를 child process에 강제한다. L1을 통과하더라도 슬로우 루프나 대용량 파일 생성을 츨다.

L3: HITL — stdin 없으면 deny‑by‑default

L1‑2를 통과한 미분류 명령은 사용자 승인을 요청한다. geode serve에서는 stdin이 없으므로 자동으로 deny‑by‑default로 처리된다.

Dual Trust Model: 입력 차단 후 출력 검증

에이전트 안전과 파이프라인 검증은 독립 축이다. Input Gating은 실행 전 차단하고, Output Validation은 실행 후 산출물을 검증한다. 두 축 모두 통과해야만 결과가 신뢰된다.

Input Gating

4‑Tier HITL 게층으로 에이전트 실행 전 차단. TIER 0‑1: asyncio.gather 자동 / TIER 3‑4: 개별 수동 승인 필수

Output Validation

5‑Tier Safety 등급으로 실행 후 출력 검증. 파이프라인 결과물만 이 아치를 통과한다

Trade-off: 3-Layer 제어 비용 L1 정적 분석 + L2 setrlimit + L3 HITL 모두 통과 시 지연이 발생하지만, 자율 에이전트가 시스템을 파괴한 사례가 0건인 것이 증거다.
