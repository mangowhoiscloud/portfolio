핵심 Trade-off 8선: 포기하지 않은 설계 결정은 결정이 아니다

TOP 3는 인과관계로 서술한다. 나머지 5개는 before/after 한 줄로 유지. 모든 피벗은 CHANGELOG에 “왔”가 기록되어 있다

TOP 1

Clean Context vs 전체 공유

Analyst A의 S급 평가가 B를 편향시키는 앙커링을 구조적으로 차단한다.

analyses 필드를 빈 배열로 전달하여 4명의 Analyst가 완전 독립 평가를 수행한다. 가헌 군사적 개입이 없는 제한된 판단이 무작위 합의보다 신뢰도가 높다.

TOP 2

DomainPort Protocol vs 하드와이어

도메인 교체 시 코어 수정 0줄. GEODE→REODE 피벗에서 실증되었다.

DomainPort가 계약만 정의하고 구현은 삽입한다. L5 엔드포인트만 교체하고 L0‑L4가 그대로 유지된다.

TOP 3

Provider-aware Compaction vs 단일 전략

Anthropic은 서버사이드(compact_20260112), OpenAI/GLM은 클라이언트사이드.

context_action.py hook이 프로바이더별 최적 전략을 분화한다. 단일 전략으로는 Anthropic의 네이티브 커패션 효율을 활용할 수 없다.

5-Tier Safety: 병렬 vs 순차

T3‑4 병렬 속도를 포기하고 DANGEROUS 명령은 개별 순차 승인으로 격리하니 권한 에스칼레이션 경로가 차단된다.

Sub-Agent 4096 Token Guard

자식 출력 전체 반환을 포기하고 4096 guard를 적용하니 부모 컨텍스트 폭발이 방지되어 MAX_CONCURRENT=5 병렬이 가능해진다.

SHA-256 PromptAssembler

수동 편집의 즉각성을 포기하고 6‑Phase 조립과 SHA‑256 추적을 도입하니 cache_control ephemeral로 2회차 입력 비용이 10%로 준다.

MCPRegistry 삭제

중앙 집중 도구 관리를 포기하고 auto‑discovery를 채택하니 proxy stub 9개가 제거되고 45 MCP 서버를 자동 등록할 수 있게 된다.

50 Rounds 후 200 Turns 상한

완전 자율을 포기하고 수렴 감지 + 비용 상한을 도입했더니 파이프라인 폭주가 차단되고 장기 운용 비용이 예측 가능해졌다.
