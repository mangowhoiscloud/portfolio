SUPPLEMENTARY: S10 THIN DAEMON

Auto-start Sequence + TOCTOU 방지 + IPC Wire Protocol

사용자가 geode를 실행하면 무엇이 일어나는가. 경합 조건 없이 안전하게 daemon을 기동하는 방법.

geode 실행 시퀀스

1

~/.geode/cli.sock 존재 확인. 있으면 연결 시도.

2

소켓 없거나 연결 실패 → pidfile lock 획득 시도.

3

lock 획득 → shutil.which("geode") serve를 subprocess.Popen으로 기동.

4

30초 timeout으로 소켓 연결 대기 (MCP 13-server 기동 ~20s).

5

연결 성공 → prompt 전송. 이후 structured events 수신 + 렌더링.

TOCTOU 방지: pidfile lock

소켓 부재 확인 → 기동 사이에 다른 프로세스가 먼저 기동하는 TOCTOU 경합을 pidfile lock(fcntl.flock)으로 차단합니다. lock 획득 실패 시 대기만 합니다.

IPC Wire Protocol (NDJSON)

Client → Server

{"type":"prompt","text":"analyze Berserk"}

{"type":"command","cmd":"/model","args":"sonnet"}

{"type":"resume","session_id":"..."}

Server → Client (30 event types)

{"type":"thinking","content":"..."}

{"type":"tool_start","name":"analyze_ip"}

{"type":"result","text":"...","rounds":3}

보안 + 에러 처리

chmod 0o600 소켓 파일 소유자만 접근 가능

listen(1) serial processing. 동시 연결 1개만 허용

BrokenPipe 클라이언트 종료 시 세션 정리

stale socket 서버 비정상 종료 시 소켓 파일 자동 정리

왜 gRPC/HTTP가 아닌가: 로컬 전용이므로 네트워크 스택 불필요. 외부 의존성 0. line-delimited JSON은 파싱 라이브러리 없이 readline()만으로 디코딩 가능합니다.
