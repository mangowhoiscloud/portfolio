SUPPLEMENTARY: S26 TRADEOFFS ARCHITECTURE

기각된 대안 상세: 왜 gRPC가 아닌지, 왜 ReAct가 아닌지

기각된 아키텍처 대안

gRPC / HTTP 서버

기각 이유: 로컬 전용 통신에 protobuf 컴파일, HTTP 서버 의존성 추가는 과잉입니다. grpcio(~40MB)는 ARM64에서 빌드 시간 3분 이상. Unix socket은 의존성 0, readline()만으로 디코딩 가능합니다.

의존성 0 vs grpcio 40MB. 선택은 명확합니다.

asyncio 이벤트 루프

기각 이유: LangGraph StateGraph는 동기 API입니다. async wrapper를 씌우면 run_in_executor 호출이 모든 노드에 필요합니다. I/O async와 Architecture async를 혼동하면 안 됩니다. 인과적 의존 체인에서 async는 복잡도만 추가합니다.

Claude Code, Codex, Devin, autoresearch 모두 동기 루프를 선택한 이유와 동일합니다.

6-Layer 유지

기각 이유: L1(Infra)과 L4(Tools)가 서로 import하는 순환 의존 12건. 6개 레이어 경계가 실제 코드 의존성과 불일치했습니다. 4-Layer로 통합하자 순환 import 0건.

기각된 파이프라인 대안

ReAct (Reasoning + Acting)

기각 이유: 파이프라인 실행 순서가 비결정적입니다. 4 analyst x 3 evaluator를 ReAct로 실행하면 순서 재현이 불가능합니다. Plan-and-Execute DAG는 결정론적 순서 + 병렬 노드를 동시에 제공합니다.

디버깅 불가능한 비결정적 파이프라인은 프로덕션에 부적합합니다.

전체 컨텍스트 공유 (analyses: full)

기각 이유: 평가자가 이전 분석 결과를 보면 앵커링 편향이 발생합니다. Clean Context(analyses: [])로 각 평가자에게 빈 상태를 주면 독립적 판단이 보장됩니다. 이 단일 결정이 평가 품질에 가장 큰 영향을 미쳤습니다.

LLM 의도 라우팅

기각 이유: 라우팅 자체에 LLM 호출 비용 + 레이턴시 + 불확실성이 추가됩니다. 정적 바인딩은 결정론적, 제로 레이턴시, 감사 가능합니다. 4개 프론티어 에이전트 중 0개가 LLM 라우팅을 사용합니다.
