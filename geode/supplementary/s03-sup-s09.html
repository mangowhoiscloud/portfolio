6-Layer에서 발생했던 의존성 문제

L1(Infra)과 L4(Tools)가 서로를 import. L2(Memory)가 L3(Orchestration)을 직접 참조. 순환 의존이 3곳에서 발생. mypy strict에서 순환 import 경고 12건.

4-Layer 전환 후

순환 import 경고 12건 → 0건. Infra/Gateway를 Runtime으로, Tools/MCP를 Harness로 통합하여 경계를 실제 의존성에 맞췄습니다.

Domain 직교 규칙: Domain은 L1(Harness)에서 DomainPort Protocol(ContextVar)로 주입됩니다. Domain이 어떤 레이어도 import하지 않고, 어떤 레이어도 Domain을 직접 import하지 않습니다.
