User explicitly rated current portfolio 4/10 overall. Key complaints:
1. "AI가 짠 것처럼 보인다" — every section follows same badge→title→description→card grid formula
2. Information density too high (0.55rem text, API-doc feel instead of portfolio)
3. No visual rhythm between sections (same dark navy throughout)
4. 13,700 lines in one HTML file
5. Hero lacks impact
6. No whitespace / breathing room

**Priority order:** 1. Section rhythm 2. Code structure 3. Hero impact

**Why:** User wants a portfolio that impresses, not documents. "포트폴리오의 역할은 감탄이지 정보 전달이 아님."
**How to apply:** Fewer sections (7-8 max), generous whitespace, alternating visual treatments, bigger typography, fewer/larger cards. Deep Sea theme colors are fine to keep.
